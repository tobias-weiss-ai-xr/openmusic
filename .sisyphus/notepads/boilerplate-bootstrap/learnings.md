# Learnings from boilerplate bootstrap
- Created GitHub boilerplate files for OpenMusic repository:
- README.md, LICENSE, CONTRIBUTING.md, CHANGELOG.md, CI workflow, issue templates, and PR template.
- Bootstrapped GitHub workflows to test TypeScript (pnpm) and Python (pytest) on ubuntu-latest.
- Directory scaffolding: created .github/workflows and .github/ISSUE_TEMPLATE as required.
- No changes were made to existing source files; this is a project bootstrap.

Important notes:
- Do not push to remote automatically; follow review process.
- Keep the README concise (<100 lines).
- Ensure templates are language-agnostic and helpful for contributions.
