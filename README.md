# OpenMusic

AI-powered dub techno generation. ACE-Step 1.5 generates raw audio segments, a Tone.js effects chain processes them with dub effects, and a Python CLI orchestrates everything into seamless mixes, short video clips, and YouTube releases.

## Quick Start

```bash
# Clone and set up ACE-Step
git clone <repo-url> openmusic && cd openmusic
git clone https://github.com/ace-step/ACE-Step-1.5.git ACE-Step-1.5

# Create ACE-Step's virtual environment
cd ACE-Step-1.5 && python -m venv .venv
source .venv/bin/activate
pip install torch transformers diffusers
cd ..

# Install the Python orchestrator into ACE-Step's venv
uv pip install -e packages/core --python ACE-Step-1.5/.venv/bin/python

# Build the TypeScript effects engine
cd packages/effects && npm install && npx tsc && cd ../..

# Generate a 10-minute mix
ACE-Step-1.5/.venv/bin/python -m openmusic.cli.main generate --length 10m --bpm 125 --key Dm --output mix.flac

# Or do the full YouTube release pipeline in one command
ACE-Step-1.5/.venv/bin/python -m openmusic.cli.main release \
  --length 2h --bpm 125 --key Dm --output mix.flac \
  --privacy unlisted --client-secrets client_secrets.json
```

## Architecture

```
                             CLI (click)
                                |
                    ┌───┬───────┴────────┬────┐
                    |   |                |    |
              generate release       short  mcp
                                      (batch-auto,
                    MixOrchestrator    generate,
                      /       \       devops-gen)
                     /         \
          ACEStepGenerator    TypeScriptBridge
          (GPU, ACE-Step)     (Tone.js effects)
                |                    |
            raw WAV             processed WAV
                \                  /
              numpy crossfade → mix.flac
                                      |
                          ┌───────────┴───────────┐
                          |                       |
                      ffmpeg MP4           short clips
                    (cover + audio)    (stoic, meditation,
                          |             devops categories)
                    YouTube Upload
```

The pipeline splits a mix into segments (default 180s). Each segment gets generated by ACE-Step, processed through dub effects, then crossfaded together. From there, `release` renders a YouTube video with cover art and uploads via OAuth, while `short` extracts clips with animated overlay content.

## CLI Usage

### Generate a mix

```
openmusic generate [OPTIONS]
  --length TEXT        Mix length (e.g. 2h, 30m, 45s)  [default: 1h]
  --bpm INTEGER        Beats per minute                 [default: 125]
  --key TEXT           Musical key                      [default: Dm]
  --output TEXT        Output file path                 [default: mix.flac]
  --effects-preset TEXT  Effects chain preset           [default: deep_dub]
  --model TEXT         Generation model (ace-step, stable-audio-open)
  --no-effects         Bypass effects processing
  --config PATH        Path to YAML config file
```

### Release to YouTube (one-shot pipeline)

```
openmusic release [OPTIONS]
  --length TEXT          Mix length                     [default: 1h]
  --bpm INTEGER          Beats per minute               [default: 125]
  --key TEXT             Musical key                    [default: Dm]
  --output TEXT          Output file path               [default: mix.flac]
  --title TEXT           Video title (auto-generated if omitted)
  --description TEXT     Video description
  --tags TEXT            Comma-separated tags
  --privacy TEXT         Visibility: public, unlisted, private  [default: public]
  --schedule TEXT        ISO 8601 premiere datetime
  --thumbnail PATH       Custom thumbnail image
  --playlist TEXT        YouTube playlist title
  --client-secrets PATH  OAuth client_secrets.json
  --no-effects           Bypass effects processing
  --model TEXT           Generation model               [default: ace-step]
```

Runs 3 steps automatically: generate mix → render MP4 with cover art → upload to YouTube.

### Short video clips

```
openmusic short generate [OPTIONS]
  Extract or generate audio + animated overlays for short-form video.

openmusic short batch [OPTIONS]
  Batch generate shorts at explicit positions from an audio file.

openmusic short batch-auto [OPTIONS]
  Auto-stagger N short clips evenly across a mix with category rotation.

  --audio PATH       Audio file to extract from
  --mix-length TEXT  Mix length (e.g. 2h, 30m, 45s)
  --count INTEGER    Number of clips              [default: 6]
  --duration INTEGER Clip duration in seconds     [default: 30]
  --categories TEXT  Comma-sep categories (stoic, meditation, devops)
  --margin INTEGER   Margin from mix edges (s)    [default: 60]

openmusic short devops-generate [OPTIONS]
  Generate a short with a devops tip overlay.

openmusic short list-authors / list-quotes
  Browse available stoic quote authors and quotes.
```

### Other commands

```
openmusic validate CONFIG_PATH   Validate YAML config
openmusic version                Print version
openmusic auth-youtube           Set up YouTube OAuth credentials
openmusic publish [OPTIONS]      Legacy all-in-one publish (mix → video → YouTube)
```

## Effects Presets

| Preset        | Delay Mix | Reverb Decay | Filter          | Character                      |
| ------------- | --------- | ------------ | --------------- | ------------------------------ |
| `deep_dub`    | 0.60      | 4.0s         | 600Hz bandpass  | Heavy, atmospheric, spacious   |
| `minimal_dub` | 0.35      | 2.0s         | 900Hz bandpass  | Clean, subtle, restrained      |
| `club_dub`    | 0.40      | 2.5s         | 1000Hz bandpass | Punchy, energetic, mid-focused |

Set via `--effects-preset` (default: `deep_dub`).

## Package Structure

```
openmusic/
├── packages/
│   ├── core/              Python orchestrator (CLI, MixOrchestrator,
│   │                      effects chain, shorts pipeline, YouTube export)
│   └── effects/           TypeScript effects engine (Tone.js)
│   └── cli/               TypeScript CLI shim (delegates to Python)
│   └── patterns/          Strudel-inspired music theory patterns
├── ACE-Step-1.5/          AI music generation model (cloned separately)
└── .sisyphus/             Sprint plans and playbooks
```

## Post-Processing

Convert to MP3:
```bash
ffmpeg -i mix.flac -c:a libmp3lame -b:a 320k mix.mp3
```

Render a YouTube video with cover art (manual):
```bash
ffmpeg -loop 1 -i cover.png -i mix.flac -c:v libx264 -tune stillimage \
  -c:a aac -b:a 256k -shortest -movflags +faststart output.mp4
```

The `openmusic release` command automates this entire flow.

## Requirements

- Python 3.12+
- Node.js (for the TS effects engine)
- [uv](https://docs.astral.sh/uv/getting-started/installation/) package manager
- ffmpeg (audio/video encoding)
- rsvg-convert (SVG cover art rendering)
- NVIDIA GPU with CUDA (recommended; CPU fallback)
- YouTube OAuth client_secrets.json (for release/publish commands)
