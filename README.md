# OpenMusic

AI-powered dub techno generation. ACE-Step 1.5 generates raw audio segments, a Tone.js effects chain runs them through delay, reverb, distortion, and vinyl simulation, and a Python CLI stitches everything into a seamless mix.

## Quick Start

```bash
# Clone and set up ACE-Step
git clone <repo-url> openmusic && cd openmusic
git clone https://github.com/ace-step/ACE-Step-1.5.git ACE-Step-1.5

# Create ACE-Step's virtual environment
cd ACE-Step-1.5 && python -m venv .venv
# Windows:
.venv\Scripts\activate
# Linux/macOS:
source .venv/bin/activate
pip install torch transformers diffusers
cd ..

# Install the Python orchestrator into ACE-Step's venv
uv pip install -e packages/core --python ACE-Step-1.5/.venv/Scripts/python.exe

# Build the TypeScript effects engine
cd packages/effects && npm install && npx tsc && cd ../..

# Generate a 10-minute mix
ACE-Step-1.5/.venv/Scripts/python.exe -m openmusic.cli.main generate --length 10m --bpm 125 --key Dm --output mix.flac
```

## Architecture

```
CLI (click)  -->  MixConfig  -->  MixOrchestrator
                                      |
                    ┌─────────────────┼─────────────────┐
                    |                 |                  |
            ACEStepGenerator   TypeScriptBridge   _assemble_segments()
                    |                 |                  |
             ACE-Step DiT      Node.js subprocess   numpy crossfade
             (GPU, 8 steps)    (Tone.js effects)    (linear blend)
                    |                 |                  |
               raw WAV          processed WAV       final mix.flac
```

The pipeline splits a target mix length into segments (default 180s each). Each segment gets generated by ACE-Step, processed through dub effects, then all segments are crossfaded together.

## CLI Usage

```
openmusic generate [OPTIONS]
  --length TEXT      Mix length (e.g. 2h, 30m, 45s)    [default: 1h]
  --bpm INTEGER      Beats per minute                   [default: 125]
  --key TEXT         Musical key (e.g. Dm, C, F#)      [default: Dm]
  --output TEXT      Output file path (.flac or .wav)   [default: mix.flac]
  --config PATH      Path to YAML config file
  --no-effects       Bypass effects processing

openmusic validate CONFIG_PATH
  Checks a YAML config has required keys: length, bpm, key, output_path

openmusic version
  Prints the current version
```

### YAML Config

```yaml
length: 2h
bpm: 128
key: Am
output_path: my_mix.flac
```

## Effects Presets

Three built-in presets control the dub processing chain:

| Preset        | Delay Mix | Reverb Decay | Filter          | Character                      |
| ------------- | --------- | ------------ | --------------- | ------------------------------ |
| `deep_dub`    | 0.60      | 4.0s         | 600Hz bandpass  | Heavy, atmospheric, spacious   |
| `minimal_dub` | 0.35      | 2.0s         | 900Hz bandpass  | Clean, subtle, restrained      |
| `club_dub`    | 0.40      | 2.5s         | 1000Hz bandpass | Punchy, energetic, mid-focused |

Set via `effects_preset` in MixConfig (default: `deep_dub`).

## Package Structure

```
openmusic/
  packages/
    core/           Python orchestrator (CLI, MixOrchestrator, ACE-Step bridge, export)
    effects/        TypeScript effects engine (Tone.js: delay, reverb, filter, distortion, vinyl)
  ACE-Step-1.5/    AI music generation model (cloned separately, own venv)
```

## Post-Processing

Convert to MP3:

```bash
ffmpeg -i mix.flac -c:a libmp3lame -b:a 320k mix.mp3
```

YouTube video with static cover image:

```bash
ffmpeg -loop 1 -i cover.jpg -i mix.flac -c:v libx264 -tune stillimage -c:a aac -b:a 256k -shortest -movflags +faststart output.mp4
```

## Requirements

- Python 3.12+
- Node.js (for the effects engine)
- [uv](https://docs.astral.sh/uv/getting-started/installation/) package manager
- ffmpeg (for MP3/FLAC encoding)
- NVIDIA GPU with CUDA (recommended; falls back to CPU)
